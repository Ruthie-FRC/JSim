# Physics Engine Documentation

Complete physics simulation system for JSim including drag, friction, collisions, and robot CAD interactions.

## Overview

The physics engine handles all forces and interactions needed for realistic FRC robot and game piece simulation:

- **Drag Forces**: Linear, quadratic, and hybrid models
- **Friction**: Coulomb, rolling, material-specific
- **Collisions**: Impulse-based dynamics with restitution
- **Aerodynamics**: Magnus effect for spinning objects
- **Robot CAD**: Mass distribution, moments of inertia, material properties
- **Integration**: Euler and Verlet methods for stable simulation

## Drag Models

### Linear Drag (Stokes Drag)

Used at low speeds in viscous fluids.

```
F_drag = -b * v
```

Where:
- `b` = linear drag coefficient (kg/s)
- `v` = velocity

**When to use:**
- Very low speed motion
- Motion in thick fluids
- Very small objects

```python
DragCalculator.calculate_linear_drag(velocity, linear_coeff=0.1)
```

### Quadratic Drag (Pressure Drag)

Dominant at moderate to high speeds in air.

```
F_drag = -0.5 * ρ * C_d * A * v * |v|
```

Where:
- `ρ` = air density (1.225 kg/m³)
- `C_d` = drag coefficient (~0.47 for sphere)
- `A` = cross-sectional area (m²)
- `v` = velocity

**When to use:**
- Typical FRC game piece flight
- Robot motion
- Most scenarios

```python
DragCalculator.calculate_quadratic_drag(
    velocity,
    drag_coefficient=0.47,
    cross_section=0.03
)
```

### Hybrid Drag

Smooth transition between linear and quadratic models.

**When to use:**
- Full speed range simulation
- More accurate predictions
- Better energy conservation

```python
DragCalculator.calculate_hybrid_drag(
    velocity,
    linear_coeff=0.1,
    drag_coefficient=0.47,
    cross_section=0.03,
    transition_speed=1.0
)
```

### Magnus Effect (Spinning Objects)

Lift force generated by spinning ball in airflow.

```
F_magnus = C_l * 0.5 * ρ * A * v * ω
```

**When to use:**
- Spinning game pieces
- Top/backspin prediction
- Advanced trajectory modeling

```python
DragCalculator.calculate_magnus_force(
    velocity,
    angular_velocity,
    magnus_coefficient=0.1,
    cross_section=0.03
)
```

## Friction Models

### Coulomb (Kinetic) Friction

Constant friction regardless of velocity.

```
f_k = μ_k * N
```

Where:
- `μ_k` = kinetic friction coefficient
- `N` = normal force

**Typical FRC coefficients:**
- Field floor (polycarbonate): 0.3-0.4
- Aluminum: 0.3
- Plastic: 0.4-0.5
- Rubber: 0.5-0.7

### Rolling Friction

Friction from rolling motion (wheels).

```
τ = μ_r * N * r
```

Typically ~5% of kinetic friction coefficient.

### Static vs Kinetic

Static friction keeps object at rest until force exceeds limit:
```
f_s,max = μ_s * N  (typically μ_s ≈ 1.1 * μ_k)
```

## Collision Physics

### Impulse-Based Resolution

```python
from physics_engine import CollisionPhysics

v1_after, v2_after = CollisionPhysics.calculate_impulse(
    mass1=50.0,        # Robot kg
    mass2=0.235,       # Game piece kg
    velocity1=(2.0, 0, 0),    # m/s (moving)
    velocity2=(0, 0, 0),      # m/s (at rest)
    restitution=0.3,          # Energy loss
    normal=(1, 0, 0)          # Collision surface normal
)
```

The method uses:
1. Relative velocity along surface normal
2. Coefficient of restitution (e)
3. Impulse calculation
4. Velocity update for both bodies

### Restitution Values

| Type | e Value | Behavior |
|------|---------|----------|
| Plastic/Inelastic | 0.0 | Stick together |
| Typical FRC | 0.3-0.5 | Some bounce |
| Bouncy | 0.7-0.9 | Significant bounce |
| Perfectly elastic | 1.0 | No energy loss |

### Moment of Inertia Calculation

For rectangular body (box):
```
I_xx = (m/12) * (width² + height²)
I_yy = (m/12) * (length² + height²)
I_zz = (m/12) * (length² + width²)
```

Parallel axis theorem for composite bodies:
```
I_total = I_cm + m * d²
```

Where `d` is distance from center of mass.

## Physics Integration Methods

### Euler Integration (Simple, Fast)

```python
from physics_engine import PhysicsIntegrator

new_pos, new_vel = PhysicsIntegrator.euler_step(
    position=(0, 0, 0),
    velocity=(1.0, 0, 0),
    acceleration=(0, -9.81, 0),
    dt=0.01
)
```

**Pros:**
- Very fast
- Easy to understand
- Works for rough estimates

**Cons:**
- Lower accuracy
- Energy drift over long simulations
- Can become unstable with large dt

### Verlet Integration (Better Energy Conservation)

```python
new_pos, new_vel = PhysicsIntegrator.verlet_step(
    position=(0, 0, 0),
    velocity=(1.0, 0, 0),
    acceleration=(0, -9.81, 0),
    dt=0.01
)
```

**Pros:**
- Better energy conservation
- More stable
- Good for long simulations

**Cons:**
- Slightly slower
- More complex

## Performance Tips

### ✓ DO

**Use appropriate time steps:**
```python
# For 50Hz robot updates
dt = 0.02  # 20ms

# For physics sub-stepping
physics_dt = 0.005  # 5ms × 4 steps per robot cycle
```

**Cache calculated values:**
```python
# Good: Pre-calculate cross-section
cross_section = math.pi * (diameter/2)**2
for _ in range(1000):
    drag = calculate_quadratic_drag(velocity, c_d, cross_section)
```

**Batch collision checks:**
```python
# Good: Check collisions once per cycle
collisions = []
for body1 in bodies:
    for body2 in bodies[bodies.index(body1)+1:]:
        if CollisionPhysics.is_collision(...):
            collisions.append((body1, body2))

# Apply all at once
for body1, body2 in collisions:
    resolve_collision(body1, body2)
```

### ✗ DON'T

**Recalculate properties every step:**
```python
# Bad: Recalculate every iteration
for _ in range(1000):
    cross_section = math.pi * (diameter/2)**2
    drag = calculate_quadratic_drag(velocity, c_d, cross_section)
```

**Use floating point comparisons:**
```python
# Bad
if speed == 0.0:
    ...

# Good
if abs(speed) < 1e-6:
    ...
```

**Process all collisions individually:**
```python
# Bad: Multiple lock acquisitions
for body1 in bodies:
    for body2 in bodies:
        if collision_detected:
            update_body1(body1)
            update_body2(body2)
```

## Example: Complete Simulation Loop

```python
from physics_engine import (
    DragCalculator, FrictionCalculator, PhysicsIntegrator,
    GRAVITY
)

# Initialize
dt = 0.01  # 10ms
mass = 0.235  # kg
velocity = (5.0, 0, 0)  # m/s
position = (0, 0, 0)

# Aerodynamic and material properties
c_d = 0.47
area = 0.03
friction_coeff = 0.4

# Simulation loop
for step in range(100):
    # Calculate forces
    acceleration_x = 0
    acceleration_y = -GRAVITY
    acceleration_z = 0
    
    # Add drag
    drag = DragCalculator.calculate_quadratic_drag(
        velocity, c_d, area
    )
    acceleration_x += drag[0] / mass
    acceleration_y += drag[1] / mass
    acceleration_z += drag[2] / mass
    
    # Add friction (horizontal surface)
    normal_force = mass * GRAVITY
    friction = FrictionCalculator.calculate_coulomb_friction(
        normal_force, friction_coeff, 
        friction_coeff * 1.1, velocity[0]
    )
    if abs(velocity[0]) > 1e-6:
        acceleration_x -= (friction / mass) * (velocity[0] / abs(velocity[0]))
    
    # Integrate
    position, velocity = PhysicsIntegrator.verlet_step(
        position, velocity,
        (acceleration_x, acceleration_y, acceleration_z),
        dt
    )
    
    print(f"Step {step}: pos={position}, vel={velocity}")
```

## Validation & Testing

### Unit Tests

```python
def test_drag_magnitude():
    """Drag force should scale with velocity²"""
    v1 = (1.0, 0, 0)
    v2 = (2.0, 0, 0)
    
    f1 = DragCalculator.calculate_quadratic_drag(v1, 1.0, 1.0)
    f2 = DragCalculator.calculate_quadratic_drag(v2, 1.0, 1.0)
    
    mag1 = math.sqrt(sum(f**2 for f in f1))
    mag2 = math.sqrt(sum(f**2 for f in f2))
    
    # f2 should be ~4x f1
    assert abs(mag2 - mag1*4) < 0.01

def test_collision_conservation():
    """Momentum should be conserved"""
    m1, m2 = 1.0, 1.0
    v1, v2 = (1.0, 0, 0), (0, 0, 0)
    
    v1_after, v2_after = CollisionPhysics.calculate_impulse(
        m1, m2, v1, v2, restitution=0
    )
    
    # Total momentum before and after
    p_before = m1*v1[0] + m2*v2[0]
    p_after = m1*v1_after[0] + m2*v2_after[0]
    
    assert abs(p_before - p_after) < 0.0001
```
